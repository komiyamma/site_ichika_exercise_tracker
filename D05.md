# 🌀 第5章：要件定義と設計 🐾

## 🌟 5.1 アプリの目的と自分の思考の始まり

運動トラッカーアプリを作る目的を考えるとき、最初に問うべきことは**「なぜこのアプリを作るのか？」**ということです。これが明確にならないと、どこから始めていいのか、何を実装すべきかがわかりません。

### **思考の軌跡**:

「私は運動が好きで、毎日の運動記録を残して後で振り返りたい。記録が簡単に入力でき、後で見返せる形で保存されると便利だな。」

ここでアプリの目的が決まります。このアプリの目標は、**「運動記録を残す」** というシンプルなものです。それができれば、後で振り返りやすく、改善にもつながります。

---

## 🌼 5.2 必要な機能の整理：アプリに必要なことを一つずつ考える

目的が決まったら、次にこのアプリに**どんな機能が必要か**を洗い出します。初心者にとっては、ここが大きな壁の一つです。まずは、「どこから始めればいいのか？」という疑問を解消するために、必要な機能をリスト化します。

### **思考の軌跡**:

「運動記録を残すには、まずその記録を追加する必要がある。次に、保存された記録を後で見ることができるようにしないといけない。もし記録を間違えたら削除できるようにする必要もある。」

このように、機能を一つずつ考えてみると、**記録の追加、表示、削除**が必要だとわかります。

---

### 必要な機能：

1. **記録追加**: ユーザーが運動の情報を入力できるフォーム
2. **記録表示**: 保存された運動記録をリストとして表示
3. **記録削除**: 不要な記録を削除する機能
4. **日付フィルター**: 特定の日付で運動記録を絞り込む機能

---

## 🌸 5.3 フォームを作成する理由と実装

フォームは、ユーザーが運動記録を**入力するため**に必要です。フォームを作成することで、**運動種目、日付、時間、回数や距離、メモ**などのデータを入力できるようになります。

### **思考の軌跡**：

「運動記録を保存するためには、まずフォームを作って、ユーザーが入力した情報を保存できるようにしないといけない。」

この段階では、フォームを作成して、ユーザーがどの情報を入力できるようにするかを決めます。**入力されたデータは後で保存**され、アプリの他の機能に利用されます。

#### **フォームに必要な情報**：

* **運動種目**（ランニング、ウォーキング、筋トレなど）
* **日付**（運動をした日）
* **時間**（運動にかけた時間）
* **回数／距離**（運動の回数や距離）
* **メモ**（自由記入欄）

これらの情報をユーザーに入力してもらうために、 **フォームが必要** です。このフォームでユーザーが記入したデータを、後で **`localStorage`に保存** する仕組みを作る準備ができます。

#### **思考の軌跡**：

「ユーザーがどんな情報を入力するのかを決め、そのデータを保存する場所（`localStorage`）に保存できるようにしないと、後で振り返る意味がない。」

---

## 🌼 5.4 保存方法を理解する：データが本当に保存されるか確認する

データを保存するために最初に使うのが、 **`localStorage`** です。これによって、入力した運動記録がブラウザに保存され、ページを再読み込みしても消えることなく残ります。

### **思考の軌跡**:

「データがどこに保存されるのか不安だな。保存した内容が見えないと、ちゃんと保存できているか不安になってしまう。」

**解決策**: 最初にデータが保存されたか確認するために、開発者ツールを使って、`localStorage`に保存した内容を直接確認します。これにより、データが正しく保存されたことを確認できるので、次に進む自信がつきます。

#### データを保存するコード：

```javascript
// データをlocalStorageに保存する関数
function saveDataToLocalStorage(data) {
  localStorage.setItem('workoutData', JSON.stringify(data)); // オブジェクトを文字列に変換して保存
  console.log("データが保存されました:", data); // 確認のため、保存されたデータを表示
}
```

* **開発者ツールで確認** : ブラウザの開発者ツール（`F12`や右クリック→「検証」）を使って、`localStorage`に保存されたデータを確認できます。

こうした方法で、**自分が入力したデータが保存された**という実感を得ることができます。この確認があると、次のステップに進む自信が持てます。

---

## 🌸 5.5 次に進む理由：フォームと`localStorage`をつなげる

フォームで入力されたデータが`localStorage`に保存されることで、データが消えることなく、アプリに永続的に保存されます。これによって、**ユーザーが運動記録を振り返ることができる**ようになります。

### **思考の軌跡**：

「保存されたデータをテーブルで表示できるようにする。表示ができれば、次に削除やフィルター機能に進める。」

ここでの思考は、「保存ができた」「確認ができた」という成功体験を持った上で、次のステップに進むことです。最初は「表示」から始めて、次に「削除」や「フィルター機能」を追加していきます。
