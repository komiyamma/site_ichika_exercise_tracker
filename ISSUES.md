# 挑戦ミッション！

このアプリを、もっと自分好みに使いやすく、もっと便利にするための「ミッション」一覧です！
`script.js` を少しずつ改造して、パワーアップさせてみましょう。

---

## MISSION 1: データの合計を計算してみよう！

今は「合計件数」しか表示されていませんが、せっかく記録した運動の時間や回数も合計できたら、もっと達成感がありそうですよね！

**ゴール:**
`renderEntryTable` 関数を改造して、絞り込まれたデータの「合計時間」と「合計回数/距離」を計算して、画面のどこかに表示してみましょう。

**ヒント:**
* `filteredEntries` という配列に、表示対象のデータが全部入っています。
* 新しい変数を2つ（`totalMinutes`, `totalValue`）用意して、`for...of` ループの中で足し算していくと計算できそうです。
* `map` を使って `minutes` だけの配列や `value` だけの配列を作ってから、合計する方法も考えられますね！

---

## MISSION 2: 新しい種目を追加してみよう！

今は選べる種目が決まっていますが、あなたがよくやる運動を追加できたら、もっと入力が楽になりますね。

**ゴール:**
`index.html` と `index2.html` を編集して、`<select id="type">` の中に、好きな種目の `<option>` を追加してみましょう。（例: `ヨガ`, `ダンス`, `ストレッチ` など）

**ヒント:**
* これはJavaScriptではなく、HTMLの編集です。
* `<option>` タグの書き方を真似すれば、簡単に追加できます。

---

## MISSION 3: `for...of` を `map` と `join` に書き換えてみよう！

`renderEntryTable` の中で、`tableHtml` という変数に `for...of` を使って一行ずつHTMLを足し算しています。
これを、`map` を使ってスタイリッシュなコードに書き換えてみましょう！

**ゴール:**
`filteredEntries` を `map` で処理し、各 `entry` から `<tr>...</tr>` のHTML文字列を作成する新しい配列を作ります。
最後に、その配列を `join('')` で連結して、一気に `entryListElement.innerHTML` にセットします。

**コードのイメージ:**
```javascript
const rows = filteredEntries.map(entry => {
  return `<tr>...（ここの中身は今までのループの中と同じ）...</tr>`;
});
entryListElement.innerHTML = rows.join('');
```

**なぜこれをするの？**
* `for` ループで文字列を足し算していくよりも、`map` で「データの配列」を「HTML文字列の配列」に変換し、最後に `join` で結合するほうが、コードがスッキリして読みやすくなります。
* `map` の練習にピッタリの課題です！

---

## MISSION 4: メモ内容で検索できるようにしよう！（高難易度）

日付だけでなく、メモの内容で検索できたら、過去の記録を探すのにとても便利になります。

**ゴール:**
* 新しい `<input type="text">` をHTMLに追加します。（絞り込みボタンの近くが良いかも？）
* その入力内容を取得し、`renderEntryTable` の中で、`entry.note` にその文字列が含まれているかどうかで絞り込む処理を追加します。

**ヒント:**
* `filter` が活躍しそうです！
* `String.prototype.includes()` というメソッドを使うと、ある文字列が別の文字列に含まれているかを `true` / `false` で判定できます。
* `(条件1 || 条件2) && 条件3` のように、複数の条件を組み合わせる必要が出てきます。
