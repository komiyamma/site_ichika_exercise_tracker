### 🎨 第6章：HTML構造と基本的なスタイリング（思考の軌跡編）

---

## 🌟 6.1 ゴールを言葉にして共有する

今回つくりたいのは「運動記録アプリ」の土台になるページです。  
でも、最初から完成形のコードを書くのではなく、**どんな見た目・どんな操作ができると嬉しいか** を言葉にしてから手を動かします。

#### 🧠 思考のメモ
- 毎日の運動を入力できるフォームが欲しい
- これまでの記録を一覧で確認したい
- まだスタイルは気にしなくていい。まずは「動く骨組み」だけ

> ゴールを文章にしておくと、「今どこまで進んだか」を振り返りやすくなります。

---

## 🗺️ 6.2 手を動かす前に構造をスケッチする

HTMLを書く前に、ページをざっくりとしたブロックに分けます。紙に四角を書いてもいいし、箇条書きでも構いません。

```
ページ全体
├─ タイトル（このページが何のためのものか伝える）
├─ 入力フォーム（運動記録を追加する場所）
└─ 記録一覧（追加した内容を眺める場所）
```

#### 🧠 思考のメモ
- ブロックの順番は「入力 → 確認」が自然そう
- 後で JavaScript からアクセスしたい場所には id を付ける予定
- Body の中で大きな塊ごとに section を使うと整理しやすい

---

## 🧱 6.3 HTML の最小スケルトンを用意する

構造が見えたら、まずは HTML の最小構成（骨組み）を作ります。  
この時点では `<body>` の中は空で構いません。

```html
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>運動記録アプリ</title>
</head>
<body>
  <!-- ここにコンテンツが入る -->
</body>
</html>
```

#### 🧠 思考のメモ
- `<!DOCTYPE html>` を忘れるとブラウザが古いモードで動くことがある
- `lang="ja"` を付けると読み上げソフトや検索エンジンが助かる
- ここまででブラウザ表示を確認しておく（白い画面でも OK）

---

## 🏷️ 6.4 ページタイトルと説明を整える

ブラウザのタブや検索結果に表示されるタイトルも早めに決めておきます。  
一言メモのような説明文があると、後から自分で見返したときに役立ちます。

```html
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>運動記録アプリ</title>
  <meta name="description" content="日々の運動量を記録して一覧表示できるシンプルなアプリの土台です。">
</head>
```

#### 🧠 思考のメモ
- `description` は必須ではないが、アプリの意図を言葉にする練習になる
- 後で SNS で共有することも想定しておくとモチベーションが上がる

---

## ✍️ 6.5 フォームに入れたい項目を決める

いきなり `<form>` を書くのではなく、何を入力したいかを文章で洗い出します。

| 欄の名前     | 目的                                   | 入力タイプの候補 |
|--------------|----------------------------------------|------------------|
| 運動種目     | どんな運動をしたかを記録する           | `text`           |
| 日付         | いつ行った運動か                       | `date`           |
| 所要時間     | どのくらいの時間トレーニングしたか     | `number` + 分    |
| 回数 / 距離  | 種目によって数字の意味を変えられるように | `number`         |
| メモ         | 感想や調子など自由に残したい情報       | `text`           |

#### 🧠 思考のメモ
- 「時間と回数」を両方入力できる UI も考えたが、まずは分けてシンプルに
- 後でバリデーションを追加しやすいように `id` を振る予定
- 必須項目は `required` で制御できるので、どこまでを必須にするか先に決めておく

---

## 🧩 6.6 フォームを最小構成で組み立てていく

フォームは塊ごとに少しずつ追加し、ブラウザで表示を確認しながら進めます。

```html
<section id="entry-form-section">
  <h2>運動記録を追加</h2>
  <form id="entry-form">
    <div class="form-field">
      <label for="type">運動種目</label>
      <input id="type" name="type" type="text" placeholder="例: ランニング" required>
    </div>

    <div class="form-field">
      <label for="date">日付</label>
      <input id="date" name="date" type="date" required>
    </div>

    <div class="form-field">
      <label for="minutes">所要時間 (分)</label>
      <input id="minutes" name="minutes" type="number" min="0" placeholder="例: 30">
    </div>

    <div class="form-field">
      <label for="value">回数 / 距離</label>
      <input id="value" name="value" type="number" min="0" placeholder="例: 回数=20 / 距離=5">
    </div>

    <div class="form-field">
      <label for="note">メモ</label>
      <input id="note" name="note" type="text" placeholder="ひとことメモ">
    </div>

    <button type="submit">記録を追加</button>
  </form>
</section>
```

#### 🧠 思考のメモ
- `<div>` でフィールドを囲むと CSS で並べ替えるときに便利
- 全て `name` 属性を付けると、後でフォームデータをまとめて取得できる
- `required` の有無でユーザー体験が変わるので、仮のルールでも決めておく

---

## 🔁 6.7 記録を表示するテーブルを用意する

入力した内容をすぐ確認できるように、テーブルも早めに置いておきます。  
この段階では中身は空ですが、後で JavaScript から行を追加しやすい形にしておきましょう。

```html
<section id="entry-table-section">
  <h2>運動記録一覧</h2>
  <table id="entry-table">
    <thead>
      <tr>
        <th>日付</th>
        <th>種目</th>
        <th>時間 (分)</th>
        <th>回数 / 距離</th>
        <th>メモ</th>
      </tr>
    </thead>
    <tbody id="entry-list">
      <!-- ここに JavaScript から行が追加される予定 -->
    </tbody>
  </table>
</section>
```

#### 🧠 思考のメモ
- `<thead>` と `<tbody>` を分けておくと、スタイリングやスクロール調整が楽
- `id="entry-list"` を付けておけば JavaScript から `document.getElementById` で取得できる
- 表の列順は後から変えられるので、まずはフォームの入力順に合わせた

## 🪄 6.8 最低限のスタイリングで読みやすくする

まだデザインには踏み込まないものの、入力欄が詰まりすぎていると試す気持ちが削がれます。  
簡単な余白とフォント設定だけ加えて、「動作確認がしやすい状態」にします。

```html
<style>
  body {
    font-family: system-ui, sans-serif;
    margin: 2rem;
    line-height: 1.6;
  }

  h1, h2 {
    margin-bottom: 0.75rem;
  }

  section {
    margin-bottom: 2rem;
  }

  .form-field {
    display: flex;
    flex-direction: column;
    gap: 0.25rem;
    margin-bottom: 1rem;
  }

  input, button {
    padding: 0.5rem;
    border: 1px solid #ccc;
    border-radius: 4px;
  }

  table {
    width: 100%;
    border-collapse: collapse;
  }

  th, td {
    border: 1px solid #ddd;
    padding: 0.5rem;
    text-align: left;
  }

  thead {
    background-color: #f5f5f5;
  }
</style>
```

#### 🧠 思考のメモ
- CSS は `<head>` の最後に書くと見つけやすい
- クラス名は「後で見返したときに役割がわかる」ことを優先
- 完璧なデザインは不要。余白と枠線だけで十分読みやすくなる
